Struktur and Files Project-Files angular.json Detailierte config für ng -- ist die Basis für alles. package.json npm-projekt-file mit "Definition"/Filter der Dependencies und Basic-Tasks package-lock.json

Nach einem lauf von npm install werden in dieser Datei alle gefundenen Abhängigkeiten incl. Download-Pfad gespeichert.

Diese Datei kann "kaputt" gehen :/ -- siehe auch das Verzeichnis "node_modules".

Löschen der Datei / des Verzeichnisses "node_modules" ist ein häufiger Bugfix, wenn man zwischen Versionen springt.

Vorlage(!) für die index.html-Datei für ng build.

Speziell base href="/" kann z.B. durch Parameter beim build verändert werden ng build --base-href=XXX

Speziell --base-href=//angular ist ein workaround für einen Git-Bash-Bug unter Windows, sonst kommt da müll raus. ( und //angular muss nach ng build manuell zu /angular korrigiert werden )
